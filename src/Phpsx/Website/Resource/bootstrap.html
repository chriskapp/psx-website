Get started

To quickly get started with PSX follow this guide which setups a working PSX environment and explains the sample project. For more detailed informations take a look at the offical documentation.

Requirements

  • PSX requires at least PHP >= 5.4 so you have to check whether you have the right version installed.

    php -v

  • PSX uses also composer as dependency managment tool. With the following command you can see whether composer is correctly installed. In case it is not installed you can use the following guide to setup composer.

    composer --version

Installation

  • Install the sample project of PSX with the following composer command. This checksout the sample project and installs a stable version of PSX.

    composer create-project psx/sample .

  • Change the file configuration.php so that the following keys are set.

    'psx_url'      => 'http://127.0.0.1:8080',
'psx_dispatch' => '',

  • Start the internal PHP web-server inside the public/ folder with the following command. Note this is only a setup for testing in production you want use a real web-server like Apache or Nginx with a vhost pointing to the public directory.

    php -S 127.0.0.1:8080 server.php

Endpoints

Now you should be able to visit the following endpoints:

  • Welcome http://127.0.0.1:8080

    The welcome API endpoint should given you an welcome message with further links to other endpoints.

  • API http://127.0.0.1:8080/internet

    This is the example API of the project.

  • Documentation http://127.0.0.1:8080/documentation/

    The automatically generated documentation of the example API.

Source code

In the following a short explanation about the important parts of the example API.

  • src/Sample/Api/InternetPopulation/Endpoint/Collection.php

    This is the class which represents the /internet endpoint. It loads the population.raml file which contains the specification of the API. Also it contains the business logic to create and receive records on an sqlite database.

  • src/Sample/Api/InternetPopulation/Resource/population.raml

    RAML specification which describes the capabilities of the API. You can add or remove parameters to change the behaviour of the API. The format of the request and response bodies is defined in the json schema files.

  • src/Sample/Dependency/Container.php

    DI container which overwrites the connection service to use an sqlite database.

  • tests/Sample/Api/InternetPopulation/Endpoint/CollectionTest.php

    Contains the PHPUnit test case for the /internet endpoint. If you have phpunit installed you can run the tests in the root of the project with the phpunit command.

Conclusion

This was a quick walk through of the PSX sample project. If you are as excited as we are about the possibilities of PSX you can dive deeper by reading the offical documentation.
